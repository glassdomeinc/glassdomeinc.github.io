Go 언어와 프로젝트 구조에 관한 이야기

November 1, 2022 · 8 min read
이현수
Senior Engineer

Go 애플리케이션의 디렉터리 구조에 관한 이야기입니다.

Go 언어와의 만남​

제가 Go를 알게 된 것은 2015년 팟캐스트 "나는 프로그래머다" 에피소드 "구글 엔지니어가 말하는 Go언어의 매력"에서였습니다. Google 엔지니어로 일했던 염재현 님이 "디스커버리 Go"라는 책을 내면서 방송에 출연해 Go 언어에 관해 이야기를 해주었습니다. 그리고 Go 스터디 그룹을 통해 튜토리얼을 조금 해보았습니다만, 그후로 사실상 실무적으로 Go를 사용한 적은 없었습니다.

글래스돔에서는 애플리케이션 개발에 Go 언어를 사용하고 있습니다.

Go 언어의 장점​

Go언어는 일단 업계에서 인기가 많습니다. 앞으로도 잘 나갈 언어라는 것입니다.

코드 문법은 어느 학교에서도 기본적으로 다 배우는 C언어를 닮았습니다.

그러면서도 C 보다 사용하기 좋습니다. (당연한 말이지만)

그리고 VM이나 스크립트 엔진 기반 언어보다 성능이 더 좋습니다.

또 제가 발견한 또 한가지 장점은,

Go 애플리케이션은 정적으로 컴파일되어 실행시간 의존성을 최소화하여 컨테이너 이미지로 빌드했을 때 크기가 아주 작게 나옵니다.

OS 조차 없는 빈 컨테이너에 Go 바이너리만 집어넣어서 실행하는 게 가능합니다. 그래서 개발하고 배포할 때 이미지 업로드 시간이 적게 들고, Kubernetes 노드에서 컨테이너 이미지를 다운로드 하는 시간과 실행 준비 시간도 적게 듭니다.

Go 컨테이너 이미지 크기

JVM이나 NodeJS가 들어간 웹 애플리케이션 서버를 빌드하면, 아무리 작게 만들려고 해도 100MB가 쉽게 넘어가고 심지어 수백 MB가 되는 경우도 있습니다. 그런데 Go 컨테이너는 필요한 패키지를 다 포함해도 10MB 미만으로 생성할 수 있습니다.

Go 설치​

tip

GVM으로 설치하는 게 좋습니다.

gvm install go1.19

Go 프로젝트 구조​

Go 문법은 쉽기 때문에 다른 프로그래밍 언어를 사용해본 개발자들은 금방 적응할 수 있습니다.

하지만 코드 외 부분들은 Go에서는 어떻게 해야하는지 궁금할 때가 있습니다.

예를 들면 Go 프로젝트의 디렉터리 구조를 어떻게 잡아야 하는 것들이 있지요.

오늘은 이에 관해 얘기해보겠습니다.

info

참고할 문서는 Standard Go Project Layout입니다.

Go 프로젝트는 모듈이다​

Go 1.14부터 Go에는 Module이라는 개념이 생겼습니다.

Go Module이 나오기 전의 Go는 Python과 같이 기본적으로 실행 환경이 설치된 곳(GOPATH)에 의존 패키지가 모여서 설치되는 구조였습니다.

info

Python에서도 마찬가지로 venv를 사용하지 않는 경우, python 설치 경로 python-x.y.z/site-packages/에 패키지가 설치됩니다.

하지만 지금에서는 옛날 이야기를 할 필요는 없고, 그냥 요즘 나오는 Go는 모두 다 Go Module이라고 보면 됩니다.

각 모듈마다 의존 패키지는 따로 관리됩니다.

info

Go Module = (다른 언어에서의) 프로젝트

Go Module 시작하기​

mymodule이라는 모듈을 만들어보겠습니다.

mkdir mymodule
cd mymodule

go mod init github.com/glassdomeinc/mymodule

go.mod 파일이 생성됩니다.

go.mod
module github.com/glassdomeinc/mymodule

go 1.19

src 디렉터리가 따로 없고, 모듈이 루트 패키지​

Java를 비롯한 JVM 계열 언어들은 컴파일러가 소스코드 파일로 인식하는 경로가 src/main/java 등으로 관습적으로 정해져있습니다.

하지만 Go에는 그런 규칙은 없는 대신, Go 모듈 디렉터리 자체가 곧 그 모듈의 루트 패키지가 됩니다.

GitHub에 호스팅된 많은 Go 오픈소스 프로젝트를 보면 저장소 자체가 Go Module이고 루트 디렉터리에 *.go 소스코드가 바로 있는 것을 확인할 수 있습니다.

+ mymodule/
+-- go.mod
+-- go.sum
+-- README.md
+-- myfile.go

이렇게 모듈 루트 디렉터리에 go.mod와 소스코드 *.go 파일을 나란히 넣어도 됩니다.

myfile.go
package mymodule

func Xxx() {
  ...
}

main 패키지(진입점 코드)은 cmd/ 디렉터리 아래에​

main.go
package main

func main() {
  ...
}

Go 애플리케이션의 진입점 코드는 main이라는 예약된 패키지에 작성합니다. 그런데 main 패키지는 다른 일반 패키지와 같은 디렉터리 내에 배치할 수 없습니다. main.go를 모듈의 루트 경로에 배치해버리면 다른 소스코드 파일이 들어갈 루트 패키지 자리(가장 중요한 곳)를 뺏는 모양이 됩니다.

그래서 main.go는 모듈의 루트 경로 대신 cmd/ 디렉터리에 배치합니다.

+ mymodule/
+-+ cmd/
| +-- main.go
+-- go.mod
+-- go.sum
+-- README.md
+-- myfile.go

(우리는 이렇게 잘 사용하지는 않지만) 만약 진입점이 여러 개면, cmd/ 아래 서브 디렉터리에 각각 배치합니다.

caution

한 디렉터리에 main 패키지는 0~1개만 존재해야 합니다.

+ mymodule/
+-+ cmd/
| +-+ main1
| | +-- main.go
| +-+ main2
|   +-- main.go
+-- go.mod
+-- go.sum
+-- README.md
+-- myfile.go
caution
