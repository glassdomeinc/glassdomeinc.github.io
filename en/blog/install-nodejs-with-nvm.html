NVM으로 NodeJS 설치하기

October 29, 2022 · 6 min read
이현수
Senior Engineer

NVM은 여러 버전의 NodeJS를 설치해두고, 필요에 따라 여러 버전을 바꿔가면서 사용할 수 있도록 도와주는 도구입니다.

TypeScript와 NodeJS​

글래스돔에서 많이 사용하는 프로그래밍 언어는 다음과 같습니다.

  • SW개발팀
    • Backend 서버 애플리케이션 개발: Go
    • Frontend 클라이언트 웹 애플리케이션 개발: TypeScript
  • Infra팀: Go, TypeScript
  • AI/Data팀: Python

그중에서 Infra팀은 클라우드 개발과 인프라 관리에 스크립트 언어가 필요한데, TypeScript를 주로 사용하고 있습니다.

TypeScript를 실행하기 위해서는 NodeJS가 필요하기 때문에 먼저 NodeJS를 설치해야 합니다.

NVM (Node Version Manager)​

NVM은 여러 버전의 NodeJS를 설치해두고, 필요에 따라 여러 버전을 바꿔가면서 사용할 수 있도록 도와주는 도구입니다.

tip

비슷하게 Go도 여러 버전을 설치하고 관리해주는 GVM이 있습니다.

이런 경우를 가정해보겠습니다.

  • AWS Lambda Function에는 Node 16 사용
  • 프론트엔드 웹 애플리케이션을 개발할 때는 Node 17 사용
  • 서버 애플리케이션에서는 Node 18을 사용

한 PC에서도 여러 버전이 설치되어야 하고, 또 때때로 버전 업그레이드나 패치도 해줘야 하고, 바꿔가면서 사용해야 하기 때문에 단순히 한 가지 버전만 설치하는 방법만으로는 매우 불편할 것입니다.

caution

그래서 NodeJS 홈페이지에 들어가서 다운로드 받아서 설치하는 방법은 추천하지 않습니다.

이럴 떄 NVM을 사용해서 NodeJS를 설치하고 버전을 관리하면 편리합니다.

설치 방법​

info

Install & Update Script

  • curl
  • wget
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.2/install.sh | bash
wget -qO- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.2/install.sh | bash

curl, wget 둘 중 아무 거나 하면 됩니다.

caution

버전은 계속 바뀌니까 그때그때 최신 버전을 확인해주세요. (2022년 10월 현재 버전 0.39.2)

쉘 환경 설정​

tip

nvm을 설치할 때 콘솔에 출력되는 방법대로 따라하면 됩니다.

  • bash
  • zsh
echo 'export NVM_DIR="$HOME/.nvm"' >> ~/.bashrc
echo '[ -s "/usr/local/opt/nvm/nvm.sh" ] && . "/usr/local/opt/nvm/nvm.sh"  # This loads nvm' >> ~/.bashrc
echo '[ -s "/usr/local/opt/nvm/etc/bash_completion.d/nvm" ] && . "/usr/local/opt/nvm/etc/bash_completion.d/nvm"  # This loads nvm bash_completion' >> ~/.bashrc
echo 'export NVM_DIR="$HOME/.nvm"' >> ~/.zshrc
echo '[ -s "/usr/local/opt/nvm/nvm.sh" ] && . "/usr/local/opt/nvm/nvm.sh"  # This loads nvm' >> ~/.zshrc
echo '[ -s "/usr/local/opt/nvm/etc/bash_completion.d/nvm" ] && . "/usr/local/opt/nvm/etc/bash_completion.d/nvm"  # This loads nvm bash_completion' >> ~/.zshrc
caution

하고 나면 터미널을 완전히 종료(⌘+Q)하고 다시 실행해주세요. 특히 VSCode가 실행중이라면, VScode를 완전히 종료하고(⌘+Q) 터미널을 재시작한 후에 VSCode를 재시작해주어야 합니다.

설치 확인​

명령줄에 아래와 같이 입력합니다.

command -v nvm

이렇게 출력되면 설치가 올바르게 된 것입니다.

nvm

.nvmrc 설정 파일​

tip

여러 프로젝트 간에 사용하는 NodeJS 버전이 엉키지 않도록 관리해주는 방법입니다.

각 프로젝트 마다 어떤 버전의 NodeJS 실행 환경을 사용해야 하는지를 .nvmrc 파일에 저장해놓고 코드 저장소에 올려놓고 사용할 수 있습니다.

lambda/.nvmrc
16.18.0
webapp/.nvmrc
17.9.1
server/.nvmrc
18.12.0

이렇게 여러 프로젝트가 있다면,

각 개발자가 프로젝트를 clone 한 다음 해당 프로젝트의 루트 경로(.nvmrc 파일이 있는 위치)에서

$ nvm install
$ nvm use

해주면 해당 NodeJS 실행 환경으로 설정됩니다.

최신 NodeJS LTS 버전 설치하기​

nvm install --lts

--lts 옵션을 붙이면 최신 LTS(Long-Term Support) 버전을 설치(혹은 업그레이드) 합니다.

note
